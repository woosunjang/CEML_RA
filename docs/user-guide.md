# 사용자 가이드

## 에이전트 가이드

### 📚 Literature Agent

논문 수집·분석·문헌 리뷰를 담당합니다. Qdrant 벡터 DB와 Scout 논문 수집 파이프라인의 데이터를 활용합니다.

**할 수 있는 것**
- 특정 주제/키워드의 논문 검색 + 분석
- 여러 논문 간 비교·대조 분석
- 연구 동향 정리 + 연구 갭 식별
- 참고문헌 기반 인용 제시

**예시 프롬프트**
1. "NASICON 고체전해질에 대한 최근 5년 연구 동향을 정리해줘"
2. "Na3Zr2Si2PO12에서 Al 도핑 효과에 대한 논문을 분석해줘"
3. "고체전해질과 황화물계 전해질의 이온전도도를 비교하는 표를 만들어줘"
4. "이 논문의 methodology가 타당한지 비판적으로 검토해줘"
5. "기계학습을 이용한 고체전해질 스크리닝 연구를 정리해줘"

**팁**
- 구체적 조성/키워드를 포함할수록 관련성 높은 결과를 반환합니다.
- "비교", "표", "분석" 키워드를 넣으면 구조화된 출력을 제공합니다.

---

### 🎓 Teaching Agent

강의 설계·교육 자료·퀴즈 생성을 담당합니다.

**할 수 있는 것**
- 특정 주제의 강의 노트 / 수업 계획 작성
- 퀴즈·시험 문제 생성 (객관식, 주관식, 계산)
- Jupyter Notebook 코드 셀 포함 교안
- 난이도별 설명 (학부 → 대학원)

**예시 프롬프트**
1. "결정 구조와 X선 회절에 대한 학부 3학년 수업 자료를 만들어줘"
2. "DFT 계산 방법론에 대한 퀴즈 5문제를 출제해줘 (난이도: 대학원)"
3. "Python으로 상평형 다이어그램을 그리는 튜토리얼을 만들어줘"
4. "이온전도도 측정 실험의 수업 교안을 작성해줘"
5. "페로브스카이트 구조를 설명하는 시각적 교안을 구성해줘"

### ✍️ Writing Agent

논문·제안서·리뷰 응답 작성을 담당합니다.

**할 수 있는 것**
- 논문 섹션별 초안 작성 (Abstract, Introduction, Methods 등)
- 연구 제안서 / 과제 신청서 작성
- 리뷰어 코멘트에 대한 응답서 작성
- 학술 영어 교정·편집

**예시 프롬프트**
1. "Na-ion 고체전해질에 대한 Introduction 섹션을 작성해줘"
2. "리뷰어가 'DFT 계산의 k-point 설정 근거가 부족하다'고 했는데 응답서를 작성해줘"
3. "이 연구 결과로 NRF 과제 신청서 초안을 작성해줘"
4. "이 Abstract를 저널 투고용 영어로 다듬어줘"
5. "Methods 섹션에 실험 조건 표를 포함하여 작성해줘"

**팁**
- 복잡한 writing 작업 (grant, rebuttal)은 자동으로 heavy 모델로 전환됩니다.
- "최종"이라는 키워드를 넣으면 Claude Sonnet이 최종 문체 편집을 수행합니다.

### 📽️ Presentation Agent

PPT·포스터·다이어그램 생성을 담당합니다.

**할 수 있는 것**
- 슬라이드 구성 + 내용 생성 (JSON 기반)
- 학술 포스터 레이아웃 설계
- 연구 흐름도 / 개념도 구조 제안
- 발표 스크립트 작성

**예시 프롬프트**
1. "Na-ion 배터리 고체전해질 연구에 대한 15분 발표 슬라이드를 구성해줘"
2. "이 논문 내용으로 학회 포스터 레이아웃을 잡아줘"
3. "연구 방법론 흐름도를 mermaid 다이어그램으로 그려줘"
4. "슬라이드 3번의 발표 스크립트를 작성해줘"

### 📋 Project Agent

과제 추적·일정 관리·리포트 생성을 담당합니다.

**할 수 있는 것**
- 프로젝트 마일스톤 / 타임라인 관리
- 회의록 정리 + 액션 아이템 추출
- 주간/월간 진행 리포트 작성
- 마감일 추적 + 우선순위 정리

**예시 프롬프트**
1. "이번 달 연구 진행 상황을 주간 리포트로 정리해줘"
2. "NRF 과제 마감까지 남은 작업을 체크리스트로 만들어줘"
3. "오늘 랩미팅 회의록을 정리해줘: [내용]"
4. "현재 진행 중인 프로젝트들의 마감일을 정리해줘"

## 특수 기능

### 🏛️ Debate 모드

3개 LLM (GPT, Claude, Gemini)이 토론하여 답변 품질을 고도화합니다.

**활성화 방법**
- API: `"mode": "debate"` 추가
- 라운드 수 조정: `"debate_rounds": 2` (기본값: 3)

**적합한 질문 유형**
- 두 가지 이상의 접근법 비교
- Trade-off 분석이 필요한 의사결정
- 기존 연구의 비판적 검토
- 가설 검증 / 메커니즘 토론

**예상 소요 시간**
- 2라운드: ~90초
- 3라운드: ~3~4분

**자동 트리거 (iMAD)**
- `auto_trigger: true` 설정 시, 복잡도 분류기가 자동으로 debate 필요 여부를 판단합니다.
- "비교분석", "trade-off", "메커니즘" 등의 키워드가 복잡도 점수를 높입니다.

### 🧠 장기 기억 (Archival Memory)

대화에서 엔티티·관계를 자동 추출하여 FalkorDB 지식 그래프에 저장합니다.

**작동 방식**
1. 대화 완료 후 백그라운드에서 자동 수집
2. "NASICON 도핑 조건은 Al 5mol%로 결정했어" → 엔티티(NASICON, Al 도핑) + 관계(결정사항) 추출
3. 다음 대화 시 관련 장기 기억을 자동 검색하여 컨텍스트에 주입

**수동 검색**
```bash
curl "http://192.168.0.12:8000/memory/search?q=도핑+조건"
```

### 📊 모델 프로필 전환

**성능 모드** (기본값): 최고 품질 모델 사용
**가성비 모드**: 비용 절감 모델 사용

```bash
# 현재 프로필 확인
curl http://192.168.0.12:8000/models/profiles

# 가성비 모드로 전환
curl -X POST http://192.168.0.12:8000/models/profile/cost

# 성능 모드로 복원
curl -X POST http://192.168.0.12:8000/models/profile/performance
```

에이전트별로 개별 전환도 가능합니다.
