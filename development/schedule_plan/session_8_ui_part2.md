## Step 1: Debate UI 전용 컴포넌트 (3h)

### 1-1. Debate 모드 토글
- 채팅 입력 바에 "🏛️ Debate" 토글 버튼 추가
- 활성화 시 입력 바 색상 변경 + "3모델 토론 모드" 안내
- `POST /chat {mode: "debate"}` 자동 전환

### 1-2. 실시간 진행 상황 UI
- SSE 기반 debate 스트리밍 수신 (세션 4에서 구현한 스트리밍 활용)
- 라운드별 패널리스트 카드:
  ```
  ┌─────────────── Round 1 ───────────────┐
  │ 🟢 analyst    ████████████ 완료 (11s) │
  │ 🟣 critic     ████████░░░░ 진행 중... │
  │ 🔵 synthesizer░░░░░░░░░░░░ 대기      │
  └───────────────────────────────────────┘
  ```
- 각 패널리스트 카드 클릭 시 해당 라운드 응답 펼침

### 1-3. Debate 결과 표시
- 최종 답변: 일반 채팅 버블과 차별화 (테두리, 배지)
- 토론 과정 접기/펼치기:
  - R1: 독립 응답 3개
  - R2: 상호 비판 3개
  - R3: 최종 입장 3개
- 메타데이터 표시: 소요 시간, 복잡도 점수

## Step 2: 채팅 UI 개선 (2h)

### 2-1. 에이전트 표시 개선
- 메시지 버블에 에이전트 아이콘 + 이름 표시
- `agent_name`에 따라 색상·아이콘 자동 배정
- "debate" 에이전트: 🏛️ 아이콘 + 보라색 테두리

### 2-2. 마크다운 렌더링 강화
- 코드 블록 구문 하이라이팅 (react-syntax-highlighter)
- 테이블 렌더링
- LaTeX 수식 렌더링 (react-katex) — 연구 맥락에서 필수
- 복사 버튼 (코드 블록 우상단)

### 2-3. 대화 관리
- 대화 목록 사이드바 (좌측)
- 새 대화 시작 버튼
- 대화 삭제
- 대화 검색 (제목/내용)

### 2-4. 모델 프로필 전환 UI
- 우상단에 프로필 스위치 (성능 ⇄ 가성비)
- 현재 프로필 표시 (뱃지)
- 전환 시 확인 토스트 알림

## Step 3: 반응형 + 접근성 (2h)

### 3-1. 반응형 디자인
- 브레이크포인트: 640px (모바일), 1024px (태블릿), 1280px (데스크톱)
- 모바일: 사이드바 → 하단 네비게이션 바
- 태블릿: 축소 사이드바 (아이콘만)
- 입력 바 고정 (모바일에서 키보드 위 고정)

### 3-2. 키보드 접근성
- Enter: 메시지 전송
- Shift+Enter: 줄바꿈
- Ctrl+K: 검색 (대화 / 장기 기억)
- Escape: 사이드바 닫기

### 3-3. 로딩 상태
- 스켈레톤 UI (대시보드)
- 타이핑 인디케이터 (채팅)
- 에러 상태 토스트
